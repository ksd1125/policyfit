3. 제품 완성도 (중간 노력)

④ 남은 258건 금액 LLM 완성 진행중

Codex 한도 소진으로 보류된 258건(미검증 167 + guard 91). 한도 회복마다 resume-amounts.py가 자동 이어서 처리. 완성 시 79% → 100% LLM 검증.

노력: 자동(한도 의존) · 효과: 전수 검증 완성

⑤ 신규 공고 자동 파이프라인 통합 중요

현재 watch-new-notices(수집)와 LLM 금액구조화가 분리. 이를 묶어 신규 공고 → 목적문 생성 → 금액 LLM 구조화 → guard → DB 병합을 단일 흐름으로. rebuild-all에 preserve 단계는 추가됨.

노력: 오케스트레이터 확장 · 효과: 운영 자동화

⑥ 매칭 고도화 선택

현재 computeMatches가 목적/업종/단계/지역 기반 pscore 산출(작동 양호). 추가로 금액규모 적합도, 마감임박 가중, 자연어 검색 의도 파싱 고려 가능.

노력: 매칭 로직 확장 · 효과: 추천 정확도 ↑
